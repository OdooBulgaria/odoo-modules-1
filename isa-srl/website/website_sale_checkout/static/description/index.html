Gestione front-end ordini ecommerce

Scopo principale della funzione

Il modulo intoduce nuovi campi durante il checkout di un ordine lato front-end dall'ecommerce. Nei dati di fatturazione viene richiesto obbligatoriamente il tipo di cliente:
  • Societá
  • Cliente privato
Se valorizzato il tipo societá vengono visualizzati e richiesti i seguenti campi:
  • Nome societá
  • Partita IVA
Se valorizzato il tipo Cliente privato vengono visualizzati e richiesti i seguenti campi:
  • Cognome
  • Codice Fiscale

Eliminazione preventivo in bozza

Se un utente effettua un ordine che poi non conferma, viene eliminato automaticamente il preventivo in stato di bozza se effettua il logout.
